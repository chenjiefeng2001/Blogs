# Multi-thread Game Engine Design

本书的目的并非深入教授游戏引擎开发，而是讲解再Direct3D渲染环境下的多线程技术。

> 补充材料：*Game Engine Architecture*

虽然该书确实构建了一个基于Direct3D且具备大多数现代便利性的游戏引擎，但目标并非尝试构建一个商业级的游戏引擎。

本书并非试图呈现一个能与商业产品或者开源引擎相媲美的尖端游戏引擎，而仅仅是一个用于演示多核线程概念的平台。我们可以在渲染其中实现许多的优化，但是目标并非构建一个典范引擎。只是一个可用的引擎。

引擎中的许多组件在本书中并未讨论。引擎构建有所涵盖，但是并非逐行、逐函数、逐类进行。出于时间和空间考虑，一些部分被刻意忽略了，尽管代码确实存在，这些功能也是已完成引擎的一部分。

本书还有一个很重要的问题：DX9、10、11有一定程度的混用。

本书基于的游戏引擎是——《Advanced 2D development game》中开发的引擎，但是显著升级了基于着色器的渲染，并且提供很多的示例。

本章主要讨论关于对称多处理器(Symmetric multiprocessing)这一整体主题。这项技术包括*多线程*——将使用这个来提高i高游戏引擎性能，这个也是本书的主要目标。

#### 深入探索SMP

当渲染和游戏本身不是瓶颈时，玩家对于一款游戏的关注点就变为对玩家的输入的相应和加载时间上

这里不会花费太多时间在探索进程间通信、线程锁场景或者线程框架内的消息传递方案。而是专注于研究线程技术本身，目的是**改进已有的游戏模式**

##### 避免为了线程而线程

将`update()`函数进行线程化一直是一个很重要的方式，但是渲染不能被线程化——帧缓冲必须是单线程

##### 先设计，后优化

本书的目标是探索在开发中端游戏引擎时基于CPU的线程处理方法，两个主题覆盖的程度大致相等。

- 一个入门级引擎基本上会封装一个SDK——DirectX之类的，并且提供基础的网格和精灵的加载与渲染，以及一个包含所有初始化、渲染和游戏逻辑代码的基本while循环。
- 一个高级的引擎会抽象渲染器，使其能够使用任何渲染SDK实现，提供场景和实体管理、用户输入、网络、GUI功能，以及通常编辑器，列举一些功能。

考虑到这些差异，因此本书开发的中端游戏引擎介于两者之间，但是更接近高级引擎。(将支持管理实体、随机等属性)

#### 多线程技术概述

你不需要一个特殊的库就可以开始探索自己的多处理器线程

非对称多线程处理是指顺序运行多个任务，或者以时间片轮转的方式运行，其中每个进程获得一些处理器时间后，便会挂起以便下一个进程。

##### 串行处理(单线程)

在一个非线程化的程序中，我们可以将其描述为串行处理程序——因为一次只能运行一个线程，仍然会存在一个while循环以及各种函数调用——因为一次运行且不重复的程序不仅罕见，并且游戏中也从未使用过。与此相反，一次性线程反而十分常见，单线程循环是当今任何语言中编写代码的最常见方式。

##### 并行处理(多线程)

在一个线程程序中，我们可以将其描述为并行处理程序——因为许多进程将同时进行，但是仍然会有一个主线程用于控制和协调其他线程的执行。
