## Part II 渲染相关

### II.1 整数维度上缩放半径

一个理想的控制多边形将节点放置在具有精确表示的位置。*当圆很小的时候可能会退化成简单的线段，并且渲染一个边缘长度不规则的凸多边形*。

> 为什么要这么做？
> 这么做有助于减少计算量——浮点数运算通常情况下会消耗掉大量的性能，而产生整数点集可以降低因此带来的大量不必要的浮点运算。

**如何找到一个半径$R$使得能够产生大量具有整数边长的三角形点集**——这类型的点集有和性质？

约分之后斜边必须是奇数长度。

*R的奇数性意味着一个斜边为偶数的三角形必须是两条偶数边/奇数边之和*——在其余情况下，两个奇数边之和给出一个斜边长度与$2 \ mod 4$同余，这并不能表示为一个完全平方数。因此，对于最简形式的三角形，偶数长度斜边被排除

通过将边长除以长斜边，可以形成具有有理系数的单位向量。此属性可以用于生成具有精确小数的坐标对，并在固定的小数位数下进行表示。

在传统的帧缓冲区上绘制一个近似精度的圆——为任何圆绘制算法提供了一个良好的京都测试，对于选择的一些整数半径，无论实现的细节如何，许多像素都应该始终被访问，因为它们精确地位于圆的周长上。

周长已知，因此查找边缘集十分简单

> 若想进一步加速可以使用
> $$
> (i^2-j^2)^2+(2i\cdot j)^2=(i^2+j^2)^2
> $$
> 来进行加速查找

### II.2 使用美观的数字创建标记

本文主要探讨如何生成美观图形标签——使用简单的方法。

在十进制中，最"好"的数字是**1，2和5**，以及这些数字的所有10次幂的倍数。我们将仅使用此类数字作为刻度间距，并在刻度间距的倍数处放置刻度标记。

对于图形范围的最大值和最小值有两种方式：

1. 松散形式：将数据最小值放在下部分，并且数据朝向最大值为上方向，以分别计算图形的最大值和最小值
2. 紧密形式：最大值和最小值分别作为图形的最大值和最小值。

### II.3 高效生成均匀缩放

本文提出了一种使用有限数量的随机数生成采样抖动的方法。所提出的抖动函数是三个变量的函数，即采样位置$(x,y)$和该位置上的编号$S$。此方法允许生成重复抖动，无需重载之前的值。并进一步应用在光线追踪算法上。

> 补充材料：
>
> 1. 分布式光线追踪 Cook 1984
> 2. 随机采样再光线追踪中的应用 Cook 1983,1984
> 3. 同样考察随机采样在光线追踪中的应用 Lee、Dippe、Wold 1985 
> 4. 随机采样作为评估“渲染方程”的基础

虽然不希望有一种方法能够在动画的每一帧中产生一组一致的样本点，但是可以希望有一种方法能够在单个帧的渲染过程中产生一组一致的样本点。解决方法目前有如下两种：

1. 将先前的结果存储起来——会带来大量的额外开销，以及IO压力
2. 使用同余随机数发生器在处理每个像素的时候生成样本点

生成一组一致的样本点不仅要求随机数函数的初始种子相同，还要求以一致的方式访问像素。(*优化算法可能会消除某些像素的需要，从而扰乱随机数序列。其他优化算法可能会改变特定像素所需的样本数量。多处理器实现会在生成一致的抖动时会遇到类似的问题*)

使用随机采样，像素区域内的采样位置会被扰动或者抖动。为了实现这种抗锯齿方法，需要一种高效的抖动生成方法。

对于数字序列重复的情况，必须确保该函数相对于图像不表现出任何空间或者时间上的规律性。

抖动函数实现：
$$
x_j=URAND(mod(x+2y+IRAND(mod(x+s,n)),n)) \\
y_j= URAND(mod(y+2x+IRAND(mod(y+s+1,n)),n))
$$
这里

- x,y=给定像素的位置
- s=给定像素的采样数
- $x_j$=
- $y_j$=
- URAND= 一个均匀随机数表
- IRAND=一个从0到n范围内的随机整数表
- n=随机数表中的元素数量

抖动函数使用两个查找表：

*一个表URAND，仅包含一组均匀分布的随机数，这些数可能被预缩放到抖动的尺寸，以便在采样过程中最终使用*。

*第二个表IRAND，包含一组整数，用作洗牌生成器，以防止函数表现出任何空间模式*——没有这个表那么生成的采样模式会重复，从而导致采样模式的相关性产生低频伪影

要么将IRAND扩展到n+1个元素并重复第一个元素，要么利用两个独立的表，一个用于x，一个用于y，来替代URAND，特别是当x和y中的抖动是缩放的而非均匀的时候。

下面是使用或运算和独立的表来对于URAND的抖动的实现：
$$
x_j= URANX(mod(or(x,2y))+IRAND(mod(x+s,n)),n) \\
y_j= URANY(mdo(or(y,2x))+IRAND(mod(y+s,n)),n)
$$

#### 估算成本

如果当前查找的表的大小为2的幂，那么模函数可以使用按位与操作代替。乘2可以使用移位操作代替。这样可以更加高效的实现——*仅需要四个假发、四个索引加载、四个按位与操作和两个移位操作*

#### 随机数生成器

典型的随机数生成器采用反馈方法，其中生成的每个数都用作生成序列中下一个数的种子。（也被称之为线性同余发生器LGG)
$$
R_{i+1}=mod(R_i s+c,m)
$$


式中的$R_{i+1}$为序列中的下一个随机数，$R_i$为当前种子，$S$、$c$和$m$则是生成器的乘数、加性项和模数项。

上述的随机数生成器的方法虽然似乎比之前的抖动函数更加简单，但是因为要用到浮点数运算反而性能并不占优，并且还因为使用多次乘法和加法，导致可重复性很差劲。

#### 在光线追踪中的应用

在光线追踪中，抖动函数用于扰动采样光线在穿过平面图像的时的方向。其他采样光线也可能被抖动。

#### 结论

本文主要探讨了一种新的抖动方法，该方法不仅计算效率高，并且产生的图像采样结果与随即抖动相当。该方法还可以用于产生重复抖动，该抖动是像素位置和样本数的函数。

### II.4 扫描转换综述

扫描转换技术是渲染二维几何图形为离散网格的一般技术。通常在图形学中，我们使用扫描转换来渲染多边形、线条以及字符串等其他此类图像元素到一个矩形网格之中。下面四篇文章主要聚焦于不同的实现用于提升速度、简单性、有效性以及支持抗锯、通用性等

#### II.4 FAA多边形扫描转换

本文介绍了一种算法——将多边形在子像素分辨率下扫描转换成光栅线，在不产生多边形到像素裁剪/暴力子像素扫描转换成本的情况下，提供了一些抗锯齿的好处。并且生成的数据还可以与深度缓冲/光栅线抗锯齿等隐藏的表面方法一起使用。

一般来说，减少计算机生成图像中的锯齿伪影的方法是将图像渲染为高分辨率，然后将生成的像素数据平均到低分辨率显示。这种方法不需要新的算法，但是为了有效防止锯齿伪影所需的额外内存和执行时间很高。

而更加精细化的隐藏表面方法，通常需要了解多边形渲染时的像素覆盖情况。提取这种子像素细节的常用方法是反复应用二维裁剪算法——将原始多边形裁剪到每个像素的边界，并计算多边形覆盖的像素面积的确切分数划分分数部分。然后使用该面积来缩放多边形颜色强度。A-buffer该方法也将剪切的多边形转换为用于像素内可见表面确定的定位掩码。

对于光滑着色器和纹理图像，在每个多边形顶点处会保留大量数据，包括模型坐标、世界坐标和法线向量以及显示坐标。所有的这些信息都必须在每个裁剪边界处进行插值——导致性能开销极高。这也意味着存在某些方法可以实现更快的近似方法，可以想办法掠过这些不必要的成本。

#### 扫描转移算法

以下算法能够高效地从顶点坐标表示的多边形中确定近似像素覆盖率。覆盖率区域和子像素掩码都可以轻易地计算出来。

> 假设要转换的多边形是凸的，且顶点是按一致顺序排列的。对于每个顶点，从转换之后的浮点数图像坐标计算整数x和y子像素坐标。实际上，每个顶点只需要存储y坐标；x坐标只引用一次，因此可以实时计算。x分辨率可以无成本的增加——最高可以达到整数的位数。而增加y分辨率需要两个整数，并且每个子像素y坐标会增加一点时间。使用2的幂操作来运算是十分方便的，似乎保持两个分辨率在同一数量级是十分明智的。为了清晰可见，伪代码假设子像素分辨率为最终渲染分辨率的八倍。

伪代码如下所示：
$$
Vertex: record [		\\
model,world,\\
normal,image:vector\\
x,y:integer;\\
];\\
Vleft,VnextLeft:Vertex;\\
Vright,VnextRight:Vertex;\\
VscanLeft,VscanRight:Vertex;\\
$$
子像素中x坐标多边形在最近的扫描线中：
$$
xLeft,xRight:array \  [0..7] \ of  \ integer
$$


在这个算法之中，**x和y指代子像素坐标**，而像素和扫描线则指代它们相应的低分辨率对应物。两者都指代显示坐标系统。多边形x和y坐标在子像素分辨率下进行插值，但是所有其他顶点信息仅在显示像素分辨率下进行插值。*为了计算像素覆盖率，只需要保存当前扫描线内每个y座标处多边形的左右x坐标*

扫描转换一个多边形。对于每条扫描线，使用子像素x边缘在八个子像素y的位置构建`xleft[]`和`xRight[]`，并插值一次其他多边形顶点信息。

从子像素中信息中渲染的多边形的一条扫描线。着色和`renderPixel`过程超出了这个Gem的范围

计算子像素位编码，指示多边形的哪些部分覆盖了像素。当多边形边缘位于像素内部时，查找表会加速掩码计算。

#### 实现细节

在多边形的第一个和最后一个扫描线上必须小心，因为某些y坐标可能不会覆盖。将未覆盖的`xLeft[]`和`xRight[]`值设置为-1足够，确保渲染扫描线能忽略这些边缘。

像素区域(和位掩码)计算可以很容易地优化以利用像素到像素的连贯性，因为扫描线内的大多数像素完全被多边形覆盖。一种方法是确定所有子像素y的最大`xLeft[]`和最小`xRight[]`，在渲染扫描线开始时。计算像素覆盖可以立即返回这些限制之间的像素最大面积。

如果像素掩码需要在比区域更粗的分辨率下计算，则应该将左右限制对每个y坐标组进行平均，以计算部分掩码。查找表可以自动计算以减少x分辨率。*如果需要掩码和区域，可以将它们一起计算以减少性能开销，或者从掩码中确定区域*

### II.5 凸多边形扫描转换和裁剪

当进行多边形着色、高洛德着色、冯氏着色或者纹理映射时，在画家算法/Z缓冲隐藏面算法红，通常需要将一组小的浮点数属性插值到多边形上。如果对所有属性使用线性插值，那么插值每个属性的代码将会非常相似，并且会因为扫描转换每个不同的属性集维护单独的源代码会变得十分麻烦。

想要减少这种麻烦程序可以通过以下三种方式：

1. 使用一个动态大小和布局的顶点结构，并且内部只包含感兴趣的属性集。
2. 使用一个包含所有属性的静态顶点结构，并对所有属性进行插值。
3. 使用一个静态的顶点结构，但是只对感兴趣的属性进行插值

方案1最省空间——但是属性偏移必须在运行时进行计算，会导致运行时很慢

方案2由于固定结构偏移允许编译时优化，因此减少了访问时间，但是整体效率依旧很低——因为未使用的属性会被分配和插值

方法3是最快的，因为在偏移编译时是固定的，并且只需要对于相关感兴趣的属性进行插值。但是相对的占用空间也是最多的一种。

使用方法3可以高效地处理各种属性集，而无需更改扫描转换器或者裁剪器的代码——因为这些例程都是与设备无关的，并且对于属性的插值一无所知。提出的算法中的扫描转换器基于Frank Crow在NYTT提出的算法，剪切器基于Ed Catmull和Alvy Ray Smith 的Sutherland-Hodgman实现。

具体实现思路如下所示：

- 将数据加载到顶点上
- 设置多边形的掩码以指示哪些属性正在使用
- 调用裁剪器修改顶点属性
- 设置多边形掩码以指示哪些属性依旧正在使用
- 调用转换器，并提供一个回调过程
- 该例程在poly_scan中的每个像素处被调用，并且在像素例程中，从插值点写在数据，使用它绘制到光栅图像中。

##### II.6 凸多边形扫描转换

多边形扫描转换比凸多边形的扫描转换更加复杂，因为每条扫描线可能与多边形在多个像素区间或者跨度内相交。

在扫描转换凹多边形时，必须跟踪当前扫描线与多边形相交的可变长度活动边列表。事实上，如果仔细研究处理条件中的不等式，则无需对这些代码来处理水平边的特殊情况。事实上，如果仔细处理条件中的不等式，则无需对这些情况进行特殊处理。由这种方法产生的程序，列在下文，出乎意料的简单。

此程序假定多边形由单个循环点序列描述。为使用此数据结构描述带孔的多边形，需构造“桥梁”将孔连接到外顶点。根据所使用的排序算法以及多边形的形状，该算法的复杂度将在$O(n)$和$O(n^2)$之间。

程序的伪代码实现如下所示：


```C++
//凹多边形：扫描转换具有n个顶点的凹非简单多边形，其顶点位于(pt[i].x,pt[i].y)，其中i在[0..n-1]的范围内，并在窗口内部通过调用`drawproc`为每个可见的像素跨度绘制。
//多边形可以是顺时针/逆时针(但是要统一)。算法在像素中心进行均匀点采样。使用约旦规则进行内部-外部测试：如果一条射线的交点次数为奇数，那么该点被认为在多边形内部，反之则在外部。
//drawproc函数应该填充扫描线y上从x.l到x.r所包含的像素
#include <vector>
#include <cmath>
#include <algorithm>

// 假设顶点结构体如下定义
struct Point {
    double x;
    double y;
};

// 外部提供的绘制扫描线跨度的函数
extern void drawproc(int y, int xl, int xr);

// 边表(ET)节点结构
struct Edge {
    int ymin;        // 边的起始扫描线
    int ymax;        // 边的终止扫描线 (包含)
    double x;        // 当前扫描线与边的交点 x 坐标
    double dx_dy;    // 扫描线每向下移动 1 像素，x 的增量 (斜率倒数)

    // 用于在活动边表 (AET) 中按 x 坐标自左向右排序
    bool operator<(const Edge& other) const {
        return x < other.x;
    }
};

/**
 * @brief 扫描转换凹非简单多边形
 * @param pt 多边形的顶点数组，依次连接
 */
void ScanFillConcavePolygon(const std::vector<Point>& pt) {
    int n = pt.size();
    if (n < 3) return;

    // 1. 获取多边形的全局极值，确定扫描线的垂直范围
    double minY = pt[0].y;
    double maxY = pt[0].y;
    for (int i = 1; i < n; ++i) {
        if (pt[i].y < minY) minY = pt[i].y;
        if (pt[i].y > maxY) maxY = pt[i].y;
    }

    // 像素中心在整数坐标。向上取整获得起始扫描线，防止越界
    // ymax 使用 ceil(maxY) - 1 避免当扫描线精确穿过顶点时，将两个相连边计算两次
    int scanMin = std::ceil(minY);
    int scanMax = std::ceil(maxY) - 1;

    if (scanMin > scanMax) return;

    // 2. 构建全局边表 (Edge Table, ET)
    std::vector<Edge> all_edges;
    for (int i = 0; i < n; ++i) {
        const Point& p1 = pt[i];
        const Point& p2 = pt[(i + 1) % n]; // 闭合多边形

        // 忽略水平边 (dx/dy 无穷大)，水平边的填充会被其它边处理
        if (p1.y == p2.y) continue;

        // 区分上下端点
        Point pBottom = p1.y < p2.y ? p1 : p2;
        Point pTop    = p1.y < p2.y ? p2 : p1;

        int ymin = std::ceil(pBottom.y);
        int ymax = std::ceil(pTop.y) - 1;

        // 如果边长不足以覆盖一个完整的扫描线像素中心区间，则跳过
        if (ymin > ymax) continue;

        double dx_dy = (pTop.x - pBottom.x) / (pTop.y - pBottom.y);
        // 计算在起始扫描线 ymin 处准确的 x 交叉点
        double start_x = pBottom.x + (ymin - pBottom.y) * dx_dy;

        Edge edge = { ymin, ymax, start_x, dx_dy };
        all_edges.push_back(edge);
    }

    // 按照起始扫描线 ymin 升序排序全局边表 (如果 ymin 相同则按起始 x 排序)
    std::sort(all_edges.begin(), all_edges.end(), [](const Edge& a, const Edge& b) {
        if (a.ymin == b.ymin) return a.x < b.x;
        return a.ymin < b.ymin;
    });

    // 3. 逐行执行扫描线转换
    std::vector<Edge> AET; // 活动边表 (Active Edge Table)
    size_t current_edge_idx = 0;

    for (int y = scanMin; y <= scanMax; ++y) {
        // a. 将在当前扫描线处新开始的边加入活动边表 AET
        while (current_edge_idx < all_edges.size() && all_edges[current_edge_idx].ymin == y) {
            AET.push_back(all_edges[current_edge_idx]);
            current_edge_idx++;
        }

        // b. 移除已经在上一行结束的边 (y > e.ymax)
        AET.erase(std::remove_if(AET.begin(), AET.end(),
            [y](const Edge& e) { return y > e.ymax; }), AET.end());

        // c. 按当前的 x 坐标交叉点自左向右对 AET 排序
        std::sort(AET.begin(), AET.end());

        // d. 使用约旦 Even-Odd 规则，将交点两两配对进行填充
        // 由于是闭合多边形，有效射线的交点数必定是偶数 (AET.size() 必定为偶数)
        for (size_t i = 0; i + 1 < AET.size(); i += 2) {
            double x_left = AET[i].x;
            double x_right = AET[i + 1].x;

            // 采样点在整数中心：左边界包含 (向上取整)，右边界不包含 (向上取整后 -1)
            // 这种标准的 Top-Left/Left-Right 封闭规则，完美实现了数学上的均匀采样
            int xl = std::ceil(x_left);
            int xr = std::ceil(x_right) - 1;

            if (xl <= xr) {
                // 绘制当前可见区间内所有像素
                drawproc(y, xl, xr);
            }
        }

        // e. 增量更新所有活动边的 x 坐标，为下一条扫描线做准备
        for (auto& edge : AET) {
            edge.x += edge.dx_dy;
        }
    }
}
```

#### II 线段绘制综述

矢量图形依旧是图形学一个重要的应用领域。虽然阴影图像有很多的优势，但是对于许多类型的图形来说，线条仍然是一种宝贵的原始元素。快速高效地绘制线条并不是一项简单的任务，特别是如果你需要避免锯齿伪影。

**目前主流的线条绘制算法为Bresenham算法**，本主题中地前三个算法会对基本的Bresenham技术进行基本介绍，随后是扩展速度或者包含抗锯齿信息。接下来的两个仅讨论线宽过宽(大于1像素)的情况，以及在矩形网格中如何高效地渲染粗线条。

#### II.8 数字线段绘制

一种数字线绘制算法是计算机图形学中最基本的工具之一，以下是Bresenham算法的骨架代码：

```C++
#include <cmath>

// 假设外部提供的画点函数
extern void drawPixel(int x, int y);

/**
 * @brief Bresenham 直线算法 (支持全方向/所有8个象限)
 * @param x0 起点 X 坐标
 * @param y0 起点 Y 坐标
 * @param x1 终点 X 坐标
 * @param y1 终点 Y 坐标
 */
void DrawLineBresenham(int x0, int y0, int x1, int y1) {
    // 1. 计算两个坐标轴上的绝对距离
    int dx = std::abs(x1 - x0);
    // 注意：dy 取负值，这是一种非常优雅的优化技巧，用来将两个维度的误差判断统一到一个计算逻辑中
    int dy = -std::abs(y1 - y0); 

    // 2. 确定步进方向 (1 或 -1)
    int sx = (x0 < x1) ? 1 : -1;
    int sy = (y0 < y1) ? 1 : -1;

    // 3. 初始化误差项
    int err = dx + dy; // 这里的本质是 dx - abs(dy)
    int e2;            // 用于存放误差项的两倍

    // 4. 开始主循环迭代绘制
    while (true) {
        // 绘制当前像素点
        drawPixel(x0, y0);

        // 如果已经到达终点，退出循环
        if (x0 == x1 && y0 == y1) {
            break;
        }

        // 记录误差项的两倍，用于接下来的判断
        // (乘以2通常会被编译器优化为左移操作 e2 = err << 1，性能极高)
        e2 = 2 * err;

        // 根据误差项决定是否在 X 轴上迈出一步
        if (e2 >= dy) {
            err += dy;  // 累加 X 步进带来的误差
            x0 += sx;   // X 向前走一步
        }

        // 根据误差项决定是否在 Y 轴上迈出一步
        if (e2 <= dx) {
            err += dx;  // 累加 Y 步进带来的误差
            y0 += sy;   // Y 向前走一步
        }
    }
}
```

#### II.9 对称双步线算法

##### 线绘制

在光栅设备上绘制直线是一个十分古老的问题，Bresenham算法作为这个问题的一个简单而有效的算法解，十分适合在硬件层面进行实现。

Bresenham算法判别式：用于比较实际线条与最近像素之间的误差

一旦确定了线条的方向，该算法在每个步骤中只需要在备选的两个像素之间做出选择。为此，需要测试判别式以找到下一个像素，然后通过一个常量进行增量，以便进行下一次测试。

二倍速Bresenham算法

该算法不使用判别式来选择下一个像素，而是使用下一个两个像素的模式。

对于一条斜线来说，一共只会有四种模式，需要在这四种模式中判别，而不是使用判别式来找。

这种利用对称性来判别并不是一个新想法——对称双步算法比原始的Bresenham算法快三到四倍。但是硬件制造商对此并不是很感兴趣，认为目前主要的瓶颈位于硬件IO上。

#### II.10 渲染抗锯齿线

问题：**如何渲染一个抗锯齿线段**

解决方案：

将线段建模为具有有限厚度，并根据每个像素与线的重叠程度设置其强度。我们通过扩展传统的Bersenham线算法来实现这一点。*在每次迭代中，通常的算法沿着主轴移动一个像素，沿着次轴移动零个或者一个像素*。为了扩展该算法，我们在传统的循环内部添加两个子循环——称为正交循环。在传统算法选择线的中心像素之后，**第一个正交循环沿着次轴的正方先检查相邻像素，然后第二个正交循环沿着次轴的负方向检查相邻像素**。

在每一个像素上，该算法更新一个变量，该变量包含像素中心与粗线中心之间的距离；该距离变量可用于计算像素与粗线的重叠程度。

使用传统的Bresenham线算法渲染几条稍微偏移的线，但使用alpha混合，并使用逐渐减少的覆盖值。这些线都应该平行，但起始位置略有不同。

#### II.11 一种用于填充二维宽线斜接缝的算法

典型的2D图形软件包允许将宽线首尾相连，其中它们的端中心点重合。倒角填充了外角之间的三角形。这是一个任意的等腰三角形。

此算法是分解斜角三角形为两个三角形并使用水平线的一种替代方法，然后生成这两个三角形，从而沿着斜角三角形的全部三条边进行Bresenham行走。此算法使用一个非旋转的直角三角形和零个、一个或者两个正交矩形填充斜角三角形区域。

优点：

- **只需要行走外边缘作为绘制直角三角形的一部分**——这样的计算量就可以减少至原来的三分之一。
- 它不会再两个内部边缘留下接缝，而旧方法可能会留下。

现在我们有两个三角形：**原始三角形(01,02,C)以及直角三角形(01,02,R)**它们共享外边缘

现在的情况可以分解为三种状态之一，具体取决于这两个三角形之间的拓扑关系。

1. C位于右三角形内部——我们只需要绘制右边的三角形
2. R位于倒角三角形内部——我们先绘制右三角形，再加上两个对角线是两条内部边的矩形
3. 两个点都不在另一个三角形内部——绘制直角三角形，再加上一个矩形，其外边是直角三角形的最长边，其对边通过C。

```c++
#include <iostream>
#include <cmath>
#include <algorithm>

// 二维点结构体
struct Point {
    int x;
    int y;
};

// ==========================================
// 外部渲染 API (需要由底层的图形或光栅化引擎提供)
// ==========================================

// 绘制非旋转（轴对齐）的直角三角形。R 为直角顶点
extern void drawRightTriangle(Point p1, Point p2, Point R);
// 绘制正交（轴对齐）矩形。p1 和 p2 是矩形的两个对角顶点
extern void drawOrthogonalRectangle(Point p1, Point p2);

// ==========================================
// 几何辅助函数
// ==========================================

// 计算二维叉积 (A->B x A->P)
// 返回值: >0 表示P在向量AB的左侧, <0 表示在右侧, =0 表示共线
// 注意：使用 1LL 防止大坐标相乘导致 int 溢出
long long crossProduct(Point A, Point B, Point P) {
    return 1LL * (B.x - A.x) * (P.y - A.y) - 1LL * (B.y - A.y) * (P.x - A.x);
}

// 判断点 P1 和 P2 是否在有向线段 AB 的同一侧
bool isSameSide(Point A, Point B, Point P1, Point P2) {
    long long cp1 = crossProduct(A, B, P1);
    long long cp2 = crossProduct(A, B, P2);
    // 只要符号相同，或者其中一点恰好在线段上，即视为在同一侧
    if (cp1 > 0 && cp2 > 0) return true;
    if (cp1 < 0 && cp2 < 0) return true;
    if (cp1 == 0 || cp2 == 0) return true;
    return false;
}

// 判断点 P 是否在三角形 ABC 的内部 (包含边界)
bool isPointInTriangle(Point A, Point B, Point C, Point P) {
    long long cp1 = crossProduct(A, B, P);
    long long cp2 = crossProduct(B, C, P);
    long long cp3 = crossProduct(C, A, P);

    bool has_neg = (cp1 < 0) || (cp2 < 0) || (cp3 < 0);
    bool has_pos = (cp1 > 0) || (cp2 > 0) || (cp3 > 0);

    // 如果不仅有正方向的叉积，也有负方向的叉积，说明点在三角形外部
    return !(has_neg && has_pos);
}

// ==========================================
// 核心算法：宽线斜接缝填充 (Miter Joint Fill)
// ==========================================

/**
 * @brief 使用直角三角形与正交矩形填充宽线接缝
 * @param O1 宽线外边缘的第一个顶点 (Outer 1)
 * @param O2 宽线外边缘的第二个顶点 (Outer 2)
 * @param C  宽线内边缘的交点 (Center / Inner point)
 */
void FillMiterJoint(Point O1, Point O2, Point C) {
    // 1. 寻找非旋转直角三角形(RAT)的直角顶点 R
    // O1 和 O2 的轴对齐包围盒构成了两个候选的直角顶点 R1 和 R2
    Point R1 = {O1.x, O2.y};
    Point R2 = {O2.x, O1.y};
    
    // 我们必须选择使得 R 和 C 位于外侧边缘线 O1-O2 同一侧的那个直角顶点
    Point R = isSameSide(O1, O2, R1, C) ? R1 : R2;

    // 2. 判断拓扑关系状态
    bool is_C_in_RAT = isPointInTriangle(O1, O2, R, C);
    bool is_R_in_Miter = isPointInTriangle(O1, O2, C, R);

    // 3. 根据描述的三种状态分配不同的光栅化指令
    if (is_C_in_RAT) {
        // 状态 1：C点位于直角三角形(O1,O2,R)内部
        // 行动：只需绘制这一个直角三角形即可完美覆盖
        drawRightTriangle(O1, O2, R);
    } 
    else if (is_R_in_Miter) {
        // 状态 2：R点位于原始斜角三角形(O1,O2,C)内部
        // 行动：绘制直角三角形，外加对角线分别是两条内部边的两个正交矩形
        drawRightTriangle(O1, O2, R);
        drawOrthogonalRectangle(O1, C); // 以 O1 和 C 为对角的矩形
        drawOrthogonalRectangle(O2, C); // 以 O2 和 C 为对角的矩形
    } 
    else {
        // 状态 3：互不包含
        // 行动：绘制直角三角形，再加一个紧贴最长直角边且延伸过C点的矩形
        drawRightTriangle(O1, O2, R);

        // 计算直角三角形两条直角边的长度
        int dx = std::abs(O1.x - O2.x);
        int dy = std::abs(O1.y - O2.y);

        Point rectCorner1, rectCorner2;
        
        // 寻找直角三角形的最长直角边，并构筑对应的正交矩形
        // 补充鲁棒性逻辑：如果 dx == dy，则依靠C点的相对位置来打破平衡，防止覆盖不到位
        if (dx > dy || (dx == dy && std::abs(C.y - R.y) > std::abs(C.x - R.x))) {
            // 水平直角边最长。矩形外边是该水平边 (y = R.y)，对边通过 C 点 (y = C.y)
            rectCorner1 = {O1.x, R.y};
            rectCorner2 = {O2.x, C.y};
        } else {
            // 垂直直角边最长。矩形外边是该垂直边 (x = R.x)，对边通过 C 点 (x = C.x)
            rectCorner1 = {R.x, O1.y};
            rectCorner2 = {C.x, O2.y};
        }
        
        drawOrthogonalRectangle(rectCorner1, rectCorner2);
    }
}
```
